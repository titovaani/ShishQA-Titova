задание 2
Каждый сайт в интернете хранится на каком-то отдельном компьютере. А каждый компьютер в свою очрередь имеет свой уникальный IP адрес.
Итак, DNS сервер . это система, которая обеспечивает связь между названием сайта и его цифровым адресом.
Главные функции сервера это:
1. хранение информации о соответсвии домена и Ip адреса, а также хранит ресурсные записи
2. кэширование данных, которое необходимо для увелечения скорости открытия нужных страниц
Этапы работы DNS сервиса
1. от пользователя поступает запрос
2. барузер отправляет его в DNS-сервис сети и ищет совпадение доменного имени и сетевого адреса.
3. как только ответ обнаружен, происходит загрузка (в случае если совпадение не обнаружено. Запрос отправляется дальше в сервис более высокого или  корневого уровней)
Корневыми серверами руководят разные операторы, которые обеспечивают бесперебойную работу сервера. На сегодня существует более ста серверов, которые располагаются в разных странах мира.

Протоколы HTTP и HTTPS используются для передачи данных в интернете. 

КЛЮЧЕВЫЕ РАЗЛИЧИЯ
В HTTP отсутствует механизм защиты для шифрования данных, в то время как HTTPS для защиты связи между сервером и клиентом использует цифровой сертификат SSL или TLS.
HTTP работает на уровне приложения, а HTTPS - на транспорном уровне.
HTTP по умолчанию работает по 80-ому порту, а HTTPS – через 443-му.
HTTP передает данные открытым текстом, а HTTPS - зашифрованным.
HTTP по сравнению с HTTPS работает быстрее, поскольку последнему нужно время для шифрования канала связи.
